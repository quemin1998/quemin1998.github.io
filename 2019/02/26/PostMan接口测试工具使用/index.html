CSDN痴乙—PostMan使用方法详解

  1. PostMan背景介绍

    用户在开发或者调试网络程序或者是网页B/S模式的程序的时候是需要一些方法来跟踪网页请求的，用户可以使用一些网络的监视工具比如著名的Firebug等网页调试工具。今天给大家介绍的这款网页调试工具不仅可以调试简单的css、html、脚本等简单的网页基本信息，它还可以发送几乎所有类型的HTTP请求！Postman在发送网络HTTP请求方面可以说是Chrome插件类产品中的代表产品之一。

  2. PostMan操作环境

    PostMan适用于不同操作系统，PostMan Mac/Windows X32/Windows X64/Linux系统，还支持PostMan浏览器扩展程序、postman chrome应用程序等。

  5. PostMan身份验证Authentication

    • Basic Auth

      是基础得验证，所以会比较简单,会直接把用户名，密码的信息放在请求的Header中(本项目所使用的就是该方法)

      1. 连接服务器后通过修改Host文件连接其他项目服务

      2. 启动项目服务后进行登陆，登陆完成后不进行退出

      3. 启动本地项目，启动本地nginx，修改Host对应项目，跳转至本地

      4. 刷新页面进行接口访问测试，是否会被拦截，或是否跳转至本地服务

        问题：登陆后访问正常，但是访问的依旧是服务器项目，并不是本地，无法连接本地代码，通过Ping访问域名对应接收服务器为本机IP

        出现该问题时刷新登陆后的服务首页，删除域名网址置灰链接单独重新访问域名，或在清除浏览器缓存后进行该操作

        若还是无法访问本地，则关闭浏览器刷新session，将浏览器重新打开后访问域名。

        可能存在项目中你本地所修改的服务模块信息不全无法登陆的情况(所以之前才需要进行Host修改)，本项目中直连本机偶尔出现登陆无法获取验证码以及登陆页面静态资源等问题

    • Digest Auth

      要比Basic Auth复杂，使用当前填写的值生成authorization header。所以在生成header之前要确保设置的正确性。如果当前的header已经存在，postman会移除之前的header。

    • OAuth 1.0

      postman的OAuth helper让你签署支持OAuth

      1.0基于身份验证的请求。OAuth不用获取access token,你需要去API提供者获取的。OAuth 1.0可以在header或者查询参数中设置value。

    • OAuth 2.0

      postman支持获得OAuth 2.0 token并添加到requests中

Postman发送请求报Could not get any response

  • 路径参数什么都是对的，还是报这个错误

  • 需要看看是否传参的方式是否正确

  • 将页面请求中Header需要填写正确之后访问才能够正常，否则报错或被拦截

  • 确保接口路径填写正确

  • 确保传参格式正确

常用的post、get、put这几个请求方式有什么不同

  • 请求方式 释义 举例

    Get GET 方法用来请求访问已被 URI 识别的资源。指定的资源经服务器端解析后返回响应内容 相当于“查看”，比如告诉服务端想查看某个主页内容，服务端就返回给你

    post 传输实体的主体 客户端：“我要把这条信息传给你”->服务端

    PUT 方法用来传输文件。就像 FTP 协议的文件上传一样，要求在请求报文的主体中包含文件内容，然后保存到请求 URI 指定的位置 客户端：“我要把这份文件传给你” ->服务端

  • HTTP协议传输返回的状态码分别代表什么意思

    有时候，接口会返回500错误，或者400,我们需要知道分别代表的什么意思

    1XX Informational（信息性状态码） 接收的请求正在处理

    2XX Success（成功状态码） 请求正常处理完毕

    3XX Redirection（重定向状态码） 需要进行附加操作以完成请求

    4XX Client Error（客户端错误状态码） 服务器无法处理请求

    5XX Server Error（服务器错误状态码） 服务器处理请求出错

Collection 运行步骤（压力）

  1. 选择collection，点击左上角runner，进入collection runner页面

  2. Collection Runner页面选择要运行的Collection，设置迭代次数，间隔时间，点击Start Test进行测试

  3. 运行结束后，查看测试结果，显示pass数量，fail数量。
